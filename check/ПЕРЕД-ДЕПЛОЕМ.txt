Перед деплоем на сервер — чеклист.

Домен
  Купить/настроить домен. A-запись на IP сервера. Для Google OAuth и куки — лучше сразу HTTPS.

Google
  Google Cloud Console → APIs & Services → Credentials → OAuth 2.0 Client ID (тип «Веб-приложение»).
  Authorized redirect URIs: https://твой-домен/api/auth/google/callback
  В .env на сервере: GOOGLE_CLIENT_ID и GOOGLE_CLIENT_SECRET.

Переменные окружения (backend/.env)
  SECRET_KEY — свой ключ (openssl rand -hex 32).
  ALLOWED_ORIGINS — https://твой-домен (или через запятую несколько).
  OPENSANCTIONS_API_KEY — если нужен поиск по санкциям.
  GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET — для входа через Google.

Сервер
  Python 3.10+, папка check целиком. Запуск из check/backend: uvicorn main:app --host 0.0.0.0 --port 8000.
  Рекомендуется: nginx как прокси, certbot для SSL (HTTPS). Бэкап файла backend/users.db.

Важно
  После смены домена — обновить redirect URI в Google Console и ALLOWED_ORIGINS в .env.
